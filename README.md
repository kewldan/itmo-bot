# ITMO Rating Bot

Telegram-бот, который следит за конкурсными списками [abit.itmo.ru](https://abit.itmo.ru)
и оценивает вероятность поступления — в моменте и в прогнозе на дату зачисления.

Мультипользовательский: каждый добавляет свои списки командой `/track`
(любые направления — бакалавриат/магистратура, контракт/бюджет) и свой код
ЕПГУ. Данные берутся из `__NEXT_DATA__` страницы списка — API и авторизация
не нужны.

## Возможности

- **Мониторинг**: опрос списков каждые 10 минут, дедупликация по `update_time`
  сайта, полная история снапшотов в PostgreSQL (JSONB).
- **Уведомления**: при каждом обновлении списка — сколько заявлений и оплат
  прибавилось, как изменилось ваше место и вероятность. Отключаются в `/list`;
  блокировка бота обрабатывается автоматически.
- **Дайджест места** (`/settings`): периодическая сводка «место, эффективная
  позиция, вероятность» не чаще выбранного интервала (выкл/3/6/12/24 ч,
  по умолчанию 6 ч). Если дайджест пора слать, он заменяет обычное
  уведомление — двух сообщений подряд не будет.
- **Детальный разбор** (`/status`): место, балл и перцентиль, разбивка
  конкурентов выше по статусам (оплатил / одобрен / согласие / без договора),
  эффективная позиция, вероятность «сейчас» и прогноз к дедлайну
  с пессимистичным/оптимистичным сценариями, скорость притока заявлений
  и оплат, прогнозы итоговых чисел, дельты за сутки.
- **Графики** (`/chart`): панель 2×2 — заявления, ваше место и эффективная
  позиция, договоры/согласия, вероятность с коридором сценариев.

## Модель вероятности

Место в списке само по себе мало значит: большинство заявлений на контракт —
«фантомы» (люди параллельно проходят на бюджет). Модель отвечает на вопрос
«сколько человек выше меня по баллу реально займут места и хватит ли мест
после них». Три слоя:

1. **В моменте.** Каждому конкуренту выше присваивается вероятность реально
   занять место по его сигналам: оплатил договор ≈ 0.97, одобрен ≈ 0.85,
   согласие ≈ 0.70, «голая» заявка — приор по приоритету программы
   (0.06–0.30). Число занятых мест выше — Пуассон-биномиальная величина;
   вероятность поступления считается точным ДП, без симуляций.
2. **Калибровка по динамике.** Приоры уточняются наблюдаемыми конверсиями
   «состояние → оплата» в истории этой же программы. Байесовское сглаживание:
   вес данных растёт и с размером когорты, и с долей прошедшего горизонта —
   12 часов без оплат не перечёркивают приор. Конверсия экстраполируется
   на оставшееся время (постоянная интенсивность), поправка к шансам
   зажата в [0.2, 5].
3. **Приток.** По истории оценивается скорость новых заявлений и доля
   новичков с баллом выше вашего; ожидаемое число новых конкурентов
   к концу приёма заявлений — Пуассон, свёртка с распределением из п. 1.

Сценарии: базовый / пессимистичный (шансы конкурентов ×2, приток ×1.6) /
оптимистичный (×0.5, ×0.5). Горизонты: `ENROLL_DEADLINE` — конец оплат,
`APPLY_DEADLINE` — конец приёма заявлений.

Для бюджетных списков используются свои приоры (согласие — сильный сигнал),
и оценка честно помечается как приближённая: модель не решает задачу
распределения по приоритетам между программами.

## Запуск (docker compose)

```bash
cp .env.example .env       # вписать BOT_TOKEN от @BotFather
docker compose up -d --build
```

В составе: бот, PostgreSQL 17 (история снапшотов), Valkey 8 (FSM-состояния).

## Разработка

Требуются [uv](https://docs.astral.sh/uv/) и Python 3.13+.

```bash
uv sync                    # окружение и зависимости (включая dev)
uv run ruff format app     # форматирование
uv run ruff check app      # линт: select = ["ALL"], без noqa
uv run ty check app        # типы
uv run python -m app.main  # локальный запуск (нужен PostgreSQL)
```

Слой БД без Docker можно проверить встроенным PostgreSQL:
`uv run --no-project --python 3.12 --with pgserver,asyncpg,httpx,pydantic-settings python <тест>`.

## Команды бота

| Команда | Действие |
|---|---|
| `/track` | подписаться: ссылка на список → код ЕПГУ |
| `/status` | детальный разбор и вероятность |
| `/chart` | графики динамики |
| `/list` | подписки: вкл/выкл уведомления, удалить |
| `/settings` | периодичность дайджеста места (выкл/3/6/12/24 ч) |
| `/refresh` | принудительно обновить данные с сайта |
| `/cancel` | отменить текущий диалог |

## Настройки (`.env`)

| Переменная | По умолчанию | Смысл |
|---|---|---|
| `BOT_TOKEN` | — | токен от @BotFather (обязательно) |
| `DATABASE_URL` | `postgresql://itmo:itmo@localhost:5432/itmo` | DSN PostgreSQL (asyncpg) |
| `REDIS_URL` | пусто | Valkey/Redis для FSM (иначе память процесса) |
| `POLL_INTERVAL_SECONDS` | 600 | период опроса сайта |
| `ENROLL_DEADLINE` | 2026-08-20 | конец приёма оплат (горизонт прогноза) |
| `APPLY_DEADLINE` | 2026-08-10 | конец приёма заявлений (горизонт притока) |

## Структура

```
app/
  config.py    настройки (.env, pydantic-settings)
  fetcher.py   загрузка и разбор __NEXT_DATA__ конкурсного списка
  metrics.py   модель вероятности: приоры, калибровка, приток, сценарии
  db.py        PostgreSQL: пул asyncpg, схема, все запросы
  models.py    датаклассы Program / Snapshot / Subscription
  service.py   сборка анализа и точек графиков из снапшотов
  charts.py    PNG-панель 2×2 (matplotlib, Agg)
  texts.py     все тексты бота (HTML)
  poller.py    фоновый опрос и рассылка уведомлений
  handlers/    aiogram-роутеры: /track (FSM) и остальные команды
  main.py      точка входа: БД, диспетчер, поллер
```
